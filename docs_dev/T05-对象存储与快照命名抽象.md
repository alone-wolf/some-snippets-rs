# T05 对象存储与快照命名抽象

## 状态

- 已完成（2026-03-29）

## 目标

建立对象存储抽象和快照文件命名规则，确保后续 draft/latest/version 写入逻辑不分散。

## 输入

- `docs_design/03-versioning-and-publishing.md`
- `docs_design/04-storage-and-data-layout.md`
- `docs_design/09-content-json-schema.md`

## 前置依赖

- `T02`
- `T03`

## 具体工作

1. 定义对象存储 trait：
   - `put_json`
   - `get_json`
   - `put_bytes`
   - `get_bytes`
2. 提供本地文件系统实现，后续可替换为 S3/OSS。
3. 定义快照路径生成器：
   - `content.draft.json`
   - `content.latest.json`
   - `content.000001.json`
4. 实现版本号格式化工具：
   - 输入整数
   - 输出 6 位零填充字符串
5. 定义 snapshot checksum 计算工具。

## 建议产物

- `src/storage/object_store/mod.rs`
- `src/storage/object_store/local.rs`
- `src/storage/snapshot/path.rs`
- `src/storage/snapshot/checksum.rs`

## 验收标准

1. 可以稳定写入和读取 JSON 文件。
2. `version=12` 时生成 `content.000012.json`。
3. draft/latest/version 的命名逻辑集中在单点实现中。

## 不在本任务内

- 业务服务调用
- 数据库事务编排
