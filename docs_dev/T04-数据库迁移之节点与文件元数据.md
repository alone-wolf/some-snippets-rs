# T04 数据库迁移之节点与文件元数据

## 状态

- 已完成（2026-03-29）

## 目标

落地 `nodes` 与 `file_metadata`，把当前设计中的节点统一模型和文件元数据模型固化到数据库。

## 输入

- `docs_design/01-domain-model.md`
- `docs_design/07-database-schema.md`

## 前置依赖

- `T03`

## 具体工作

1. 创建 `nodes` 表：
   - `id`
   - `content_id`
   - `uuid`
   - `version`
   - `kind`
   - `lifecycle_state`
   - `text_content`
   - `prev_node_id`
   - `meta_json`
   - 审计字段
2. 创建 `file_metadata` 表：
   - `node_id`
   - `file_uuid`
   - `bucket`
   - `object_key`
   - `filename`
   - `mime_type`
   - `size_bytes`
   - `checksum`
   - `meta_json`
3. 建立关键索引：
   - `nodes(uuid, version)`
   - `nodes(content_id, lifecycle_state)`
   - `nodes(prev_node_id)`
   - `file_metadata(node_id)`
4. 定义枚举或受限字符串：
   - `kind = text | file`
   - `lifecycle_state = draft_only | committed`

## 建议产物

- `migration/src/m20xxxxxx_create_nodes.rs`
- `migration/src/m20xxxxxx_create_file_metadata.rs`
- 对应 SeaORM entity

## 验收标准

1. `(uuid, version)` 唯一约束生效。
2. `prev_node_id` 可自引用。
3. `file_metadata.node_id` 可稳定关联 file 节点。

## 不在本任务内

- 节点编辑业务逻辑
- 文件上传逻辑
