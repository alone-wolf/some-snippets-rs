# T12 权限校验与审计接入

## 状态

- 已完成（2026-03-29）

## 目标

把首版 RBAC 接入到内容和节点操作链路中，避免先实现所有能力后再整体返工。

## 输入

- `docs_design/02-permission-and-policy.md`
- `docs_design/05-api-design.md`

## 前置依赖

- `T11`

## 具体工作

1. 定义首版权限枚举：
   - `content:read`
   - `content:update`
   - `content:commit_latest`
   - `content:create_version`
   - `content:rollback`
   - `node:create`
   - `node:update`
2. 设计用户上下文注入方式。
3. 在关键 handler 前增加权限检查中间件或 service 校验。
4. 记录关键操作审计字段：
   - `created_by`
   - `updated_by`
   - version 创建人
5. 明确 `node:update` 在 committed 场景下等价于“创建替代节点并更新 draft”。

## 建议产物

- `src/modules/auth/permission.rs`
- `src/web/middleware/authz.rs`
- 相关 service 的审计字段填充逻辑

## 验收标准

1. 无权限请求会被拒绝。
2. commit/version/rollback 都有独立权限点。
3. 审计字段能记录发起用户。

## 不在本任务内

- 多租户组织模型
- 复杂策略引擎
