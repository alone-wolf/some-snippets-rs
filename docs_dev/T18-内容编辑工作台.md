# T18 内容编辑工作台

## 状态

- 已完成（2026-03-29）

## 目标

实现 `web-admin` 的核心编辑工作台，承接 draft 编排、节点编辑与 copy-on-write 交互。

## 输入

- `docs_design/10-web-frontend-and-hosting.md`
- `docs_design/03-versioning-and-publishing.md`
- `docs_design/09-content-json-schema.md`

## 前置依赖

- `T16`
- `T17`

## 具体工作

1. 实现 draft 加载与展示。
2. 以可视列表展示当前节点顺序。
3. 支持新建 text 节点。
4. 支持新建 file 节点（首版可先按元数据录入模式）。
5. 支持编辑 text / file 节点。
6. 支持节点顺序调整。
7. 支持节点替换后的本地状态刷新。
8. 明确展示节点生命周期：
   - `draft_only`
   - `committed`
9. 对 committed 节点编辑时，提供 copy-on-write 提示：
   - 编辑将创建替代节点
   - draft 引用将切换为新节点
10. 建立编辑工作台的局部状态：
    - `serverSnapshot`
    - `workingState`
    - `dirty`
    - `saving`

## 建议产物

- `web-admin/src/views/editor/*`
- `web-admin/src/components/node/*`
- `web-admin/src/composables/useDraft.ts`
- `web-admin/src/stores/editor.ts`

## 验收标准

1. 可以查看当前 draft 节点顺序。
2. 可以新增、编辑、调整节点。
3. committed 节点的编辑提示与结果符合 copy-on-write 语义。
4. 编辑工作台能区分服务端状态与界面临时状态。

- 大文件上传优化
