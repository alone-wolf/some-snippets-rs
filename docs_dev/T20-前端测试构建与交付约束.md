## 具体工作

1. 配置前端 lint。
2. 配置前端单元测试框架：
   - `vitest`
   - `vue-test-utils`
3. 为关键模块补最小测试：
   - 路由渲染
   - API client 基础行为
   - 核心页面组件基础渲染
4. 明确构建输出目录为 `web-admin/dist`。
5. 确保构建产物可被 `axum` 托管。
6. 增加开发文档：
   - 如何安装依赖
   - 如何本地开发
   - 如何构建
7. 明确 CI / 本地建议命令：
   - `npm run lint`
   - `npm run test`
   - `npm run build`

## 建议产物

- `web-admin/package.json`
- `web-admin/tests/*`
- `web-admin/README.md`

## 验收标准

1. `web-admin` 可独立完成 lint、test、build。
2. 构建产物与 `axum` 托管路径一致。
3. 至少具备最小自动化覆盖，而不只是纯手工验证。
4. 开发者可以按文档完成前端启动与构建。
