## 状态

## 目标

用自动化测试把当前设计中最关键、最容易回归的链路固定下来。

## 具体工作

1. 建立测试数据库夹具。
2. 建立本地对象存储测试夹具。
3. 覆盖以下关键用例：
   - draft_only text 节点原地更新
   - committed text 节点 copy-on-write
   - committed file 节点 copy-on-write
   - draft -> latest
   - latest -> version
   - version -> rollback -> draft
4. 校验 snapshot schema：
   - draft 只含 `nodeId`
   - latest/version 含完整节点快照
5. 校验版本命名：
   - JSON 中 version 为整数
   - 文件名为 6 位零填充

## 验收标准

1. 关键版本流转链路有自动化覆盖。
2. copy-on-write 规则有测试覆盖。
3. rollback 的归一化 draft 结果有测试覆盖。
