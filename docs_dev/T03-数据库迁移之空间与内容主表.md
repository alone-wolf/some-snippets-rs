# T03 数据库迁移之空间与内容主表

## 状态

- 已完成（2026-03-29）

## 目标

落地 `collections`、`contents`、`content_versions` 三张主表，为后续内容管理和版本索引提供基础。

## 输入

- `docs_design/01-domain-model.md`
- `docs_design/07-database-schema.md`

## 前置依赖

- `T02`

## 具体工作

1. 新建 migration 工程或 migration 模块。
2. 创建 `collections` 表：
   - `id`
   - `slug`
   - `name`
   - `description`
   - `visibility`
   - `owner_id`
   - `config_json`
   - 时间字段
3. 创建 `contents` 表：
   - `collection_id`
   - `slug`
   - `title`
   - `status`
   - `schema_id`
   - `draft_snapshot_key`
   - `latest_snapshot_key`
   - `latest_version`
4. 创建 `content_versions` 表：
   - `content_id`
   - `version`
   - `label`
   - `snapshot_key`
   - `snapshot_checksum`
   - `meta_json`
5. 加上必要唯一索引和外键。

## 建议产物

- `migration/src/m20xxxxxx_create_collections.rs`
- `migration/src/m20xxxxxx_create_contents.rs`
- `migration/src/m20xxxxxx_create_content_versions.rs`
- 对应 SeaORM entity

## 验收标准

1. migration 可以成功执行。
2. `(content_id, version)` 唯一约束生效。
3. `latest_version` 是整数，并具备默认值 `0`。

## 不在本任务内

- `nodes`
- `file_metadata`
- 快照写入逻辑
