# T19 版本管理页面

## 状态

- 已完成（2026-03-29）

## 目标

在 `web-admin` 中提供版本流转页面，打通 commit latest、create version、版本浏览与 rollback 交互。

## 输入

- `docs_design/10-web-frontend-and-hosting.md`
- `docs_design/03-versioning-and-publishing.md`
- `docs_design/05-api-design.md`

## 前置依赖

- `T16`
- `T18`

## 具体工作

1. 实现版本列表页。
2. 实现 latest 快照查看入口。
3. 实现 version 快照查看页。
4. 实现 `commit latest` 交互：
   - 确认弹窗
   - 成功反馈
5. 实现 `create version` 交互：
   - 可选填写 `label`
   - 成功后刷新版本列表
6. 实现 `rollback` 交互：
   - 明确提示“恢复为 draft，不直接覆盖 latest”
   - 执行后跳转或刷新编辑工作台
7. 展示版本号、label、checksum、snapshot key 等核心信息。

## 建议产物

- `web-admin/src/views/versions/*`
- `web-admin/src/components/version/*`
- `web-admin/src/composables/useVersioning.ts`

## 验收标准

1. 可以在页面中完成 commit latest。
2. 可以创建 version 并查看版本列表。
3. 可以查看指定 version 快照。
4. rollback 的提示与行为符合设计文档语义。

## 不在本任务内

- 审批流
- 复杂版本对比视图
- 回滚后的自动再次提交
