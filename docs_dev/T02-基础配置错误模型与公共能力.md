# T02 基础配置、错误模型与公共能力

## 状态

- 已完成（2026-03-29）

## 目标

建立后续任务都会依赖的公共基础设施，避免在每个模块里重复定义配置、错误和上下文。

## 输入

- `docs_design/04-storage-and-data-layout.md`
- `docs_design/06-service-architecture-axum.md`

## 前置依赖

- `T01`

## 具体工作

1. 设计应用配置结构：
   - `server`
   - `database`
   - `object_store`
   - `auth`
2. 增加环境变量加载逻辑。
3. 统一错误模型：
   - 领域错误
   - 存储错误
   - 对外 API 错误映射
4. 定义应用上下文 `AppState`：
   - db 连接
   - object store client
   - 配置
5. 提供统一响应包装和日志埋点入口。

## 建议产物

- `src/config/mod.rs`
- `src/error/mod.rs`
- `src/app/state.rs`
- `src/web/response.rs`

## 验收标准

1. 服务可通过环境变量加载配置。
2. 业务错误可以稳定映射到 HTTP 状态码。
3. `AppState` 可被 handler 和 service 共享。

## 不在本任务内

- 实际业务表结构
- 权限细节
