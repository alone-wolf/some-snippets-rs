# some-snippets

A small Axum + SeaORM service that exposes generic CRUD APIs for snippet-related resources.

## Run

```bash
cargo run
```

Default bind: `127.0.0.1:3000`  
Default database: `sqlite://snippets.db?mode=rwc` (override with `DATABASE_URL`).
Default upload directory: `uploads` (override with `FILE_STORAGE_DIR`).

### Recommended dev start script

```bash
./scripts/dev-start.sh
```

It will:

- sanitize unsupported sqlite URL params (like `foreign_keys`)
- apply migrations
- check whether port `3000` is already occupied
- start the backend server

During backend startup, `FILE_STORAGE_DIR` is validated; if it does not exist, the service tries to create it and exits on failure.

If you only want preflight checks:

```bash
./scripts/dev-start.sh --check-only
```

## Web Panel (Vue3 + Element Plus)

```bash
cd web
npm install
npm run dev
```

Default panel URL: `http://127.0.0.1:5173` (dev proxy to backend `127.0.0.1:3000`).

## API

- `GET /ping`
- CRUD (under `/api/v1`):
  - `/collections`
  - `/files`
  - `/histories`
  - `/nodes`
  - `/snippets`
  - `/tags`
  - `/texts`

All endpoints use the no-trailing-slash style (for example, `/api/v1/tags` and `/api/v1/tags/:id`).

Resource APIs:

- Common resources (except `collections`):
  - `GET /api/v1/<resource>`
  - `POST /api/v1/<resource>`
  - `GET /api/v1/<resource>/:id`
  - `PUT /api/v1/<resource>/:id`
  - `DELETE /api/v1/<resource>/:id`
- Collections:
  - `GET /api/v1/collections`
  - `POST /api/v1/collections`
  - `GET /api/v1/collections/:key`
  - `PUT /api/v1/collections/:key`
  - `DELETE /api/v1/collections/:key`
- File upload:
  - `POST /api/v1/files/upload` (multipart field name: `file`)

List APIs (`GET /api/v1/<resource>`) support pagination query params:

- `page` (default: `1`)
- `page_size` (default: `20`, clamped to `1..=200`)
- list results are ordered by primary key ascending for stable pagination

### Node payload rules

For `/api/v1/nodes`:

- `kind` is required and must be `text` or `file`
- `kind=text` requires `text_id` and forbids `file_id`
- `kind=file` requires `file_id` and forbids `text_id`

## Notes

- Tables are created by SeaORM migration in `migration/src/m20260215_000001_create_tables.rs`.
- Collection fields include:
  - `label`: display name
  - `key`: unique URL path segment
- Uploaded files are persisted as UUIDv4 filenames without suffix; the UUID is stored in `files.file_uuid`.
- Database-level foreign key constraints are enabled by migration.
- Reference integrity is also validated in the data access layer during create/update.
- Write operations use transactions so validation and writes are executed atomically.
- Delete operations verify target existence; reference conflicts are enforced by database foreign keys.
- `created_at` / `updated_at` are generated by the server; clients should not send them.
